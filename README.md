## What is QuestForge?

QuestForge turns dry **System Design interview questions** into retro fantasy RPG quests.

You pick a challenge (or type your own). The app calls Google Gemini to generate:
- ⚔️ A **quest title** in the style of a game level
- 📜 A **fantasy narrative** that maps system design concepts to fantasy elements
- 💡 A **real architectural lesson** hidden inside the story
- 🎨 A **pixel-art scene** — AI-generated by Pollinations.ai, no API key needed

Every quest teaches you something real about system design — but feels like playing a game.

## How the AI Works

### Groq + Llama 3.1 — Text Generation

QuestForge uses the **Groq API** with **Llama 3.1 8B Instant** for all quest narrative generation.

```
POST https://api.groq.com/openai/v1/chat/completions
Authorization: Bearer YOUR_GROQ_KEY
```

**Why Groq?**

| Reason | Detail |
|--------|--------|
| ✅ **Genuinely free** | 14,400 requests/day, no credit card, no billing — works everywhere |
| ✅ **No region restrictions** | Available globally, unlike Google Gemini which blocks the free tier in many regions |
| ✅ **Lightning fast** | Groq's custom LPU hardware — responses arrive in under 1 second |
| ✅ **Reliable JSON output** | `response_format: json_object` forces clean structured JSON every time |
| ✅ **OpenAI-compatible API** | Standard format, easy to understand and maintain |

The prompt instructs Llama to map system design concepts to fantasy metaphors:

```
Rate Limiter     → The Gatekeeper's Toll
Load Balancer    → The Twin Sorcerers
Cache            → The Crystal Memory Vault
Database         → The Ancient Tome of Records
Message Queue    → The Messenger Ravens
Docker Container → The Summoning Circle
CDN              → The Network of Mirror Portals
Circuit Breaker  → The Failsafe Seal
```

The model returns structured JSON with `title`, `story`, `lesson`, and `image_scene` — parsed directly in the browser. If the primary model hits a rate limit, the app automatically falls back to `gemma2-9b-it` and then `llama3-8b-8192`.

### Pollinations.ai — Pixel-Art Image Generation

The **quest illustration** is generated by [Pollinations.ai](https://pollinations.ai) — a completely free, no-key image generation API.

```js
const url = `https://image.pollinations.ai/prompt/${encodedPrompt}?width=832&height=448&model=flux&nologo=true&seed=${Date.now()}`;
```

No API key. No billing. The URL itself is the request. The image loads directly into the `<img>` tag.

The prompt is built from the Gemini quest output, requesting a **16-bit pixel-art RPG scene** with dark purple backgrounds and neon glow effects — matching the app's visual style perfectly.

## The Journey: Claude → Gemini → Groq

QuestForge **started with the Anthropic Claude API** (`claude-sonnet-4-6`) for its strong narrative reasoning. When the credit balance ran out it returned:
```
Error 400: Your credit balance is too low to access the Claude API.
```

Switched to **Google Gemini** (free tier), but hit a second wall — Gemini's free tier quota is `limit: 0` for many API keys depending on your Google Cloud region and project settings, making it unreliable for a public project anyone can use:
```
Error 429: Quota exceeded — limit: 0, model: gemini-2.0-flash
```

Switched to **Groq** — genuinely free (14,400 req/day), no region restrictions, no credit card, and actually faster than both. For a portfolio project you want *anyone* to try without setup friction, Groq is the right call.

Each migration was a one-file change — the architecture is identical: send a prompt, get JSON back.

## How the Project Was Deployed

This entire project was built and deployed using **Claude Code** (Anthropic's AI coding assistant) without ever touching the GitHub web UI.

**Steps taken, all via terminal/API:**

1. **Created the GitHub repository** using the GitHub REST API:
   ```bash
   curl -X POST https://api.github.com/user/repos \
     -H "Authorization: token $GITHUB_TOKEN" \
     -d '{"name":"questforge","private":false}'
   ```

2. **Enabled GitHub Pages** via the API:
   ```bash
   curl -X POST https://api.github.com/repos/mayageva11/questforge/pages \
     -H "Authorization: token $GITHUB_TOKEN" \
     -d '{"source":{"branch":"main","path":"/"}}'
   ```

3. **Pushed the code** using git with inline credential helper (never stored the token in `.git/config`)

4. **Live in ~2 minutes** at `https://mayageva11.github.io/questforge`

Claude Code acted as a **GitHub automation layer** — what some call "GitHub MCP" (Model Context Protocol integration) — orchestrating the entire deployment pipeline from prompt to live URL without manual browser clicks.

## Local Setup

```bash
git clone https://github.com/mayageva11/questforge.git
cd questforge
open index.html   # macOS
# or just drag index.html into your browser
```

Then:
1. Get a **free** Groq API key at [console.groq.com](https://console.groq.com) (takes ~30 sec, no credit card, no billing)
2. Click the ⚙️ gear icon → paste your key → Save
3. Pick a challenge and hit **Forge Quest ⚔️**

Your API key is stored only in your browser's `localStorage`. It goes directly from your browser to Groq — never touches any other server.

---

## Deploy Your Own

1. **Fork** this repo
2. Go to **Settings → Pages**
3. Set source to **Deploy from branch → main → / (root)**
4. Wait ~2 minutes → your copy is live at `https://YOUR_USERNAME.github.io/questforge`

No configuration needed. No environment variables. The user brings their own Gemini key.

## Design

Glassmorphic dark UI with a fantasy-RPG aesthetic:

- **Color palette:** Deep purple `#06040b` background, pink `#f472b6`, lavender `#a78bfa`, sky blue `#38bdf8`, amber `#fbbf24`
- **Glitch animation** on the app title — fires once on load
- **Shimmer skeleton** while the quest image loads
- **View Transitions API** for smooth screen-to-screen navigation (with fallback for Safari)
- **Safari-hardened** modal system — triple-layer hide (`display: none !important`, `visibility: hidden`, inline style) to prevent bleed-through
